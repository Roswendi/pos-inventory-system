HOW IT WORKS (Technical)
================================================================================

The connection happens through these files:

1. manifest.json
   - Defines app name, icons, start URL
   - Links logo192.png and logo512.png
   - Tells browser this is an installable app

2. index.html
   - References manifest.json
   - Links to service worker
   - Sets up PWA

3. Service Worker (sw.js)
   - Enables offline functionality
   - Makes app work like native app

4. Icons (logo192.png, logo512.png)
   - Used when installing
   - Displayed on home screen
   - Clicking opens the app

All of this is ALREADY SET UP! You just need to:
- Add your icons
- Build the app
- Deploy it
- Install it

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Icon doesn't appear when installing
----------------------------------------------
SOLUTION:
- Check icons exist: ls frontend/public/logo*.png
- Check manifest.json references icons correctly
- Rebuild: npm run build
- Clear browser cache
- Try different browser

PROBLEM: Clicking icon doesn't open app
----------------------------------------
SOLUTION:
- Make sure app is deployed (not just local)
- Check backend is running and accessible
- Verify API URL is correct
- Check browser console for errors
- Reinstall the app (remove and add again)

PROBLEM: "Add to Home Screen" option not showing
-------------------------------------------------
SOLUTION:
- Must use HTTPS (not HTTP)
- Must be deployed online (not localhost)
- Use Chrome (Android) or Safari (iOS)
- Check manifest.json is accessible
- Clear browser cache

PROBLEM: App opens but shows errors
-----------------------------------
SOLUTION:
- Check backend is running
- Verify API URL in frontend/src/services/api.js
- Check browser console for errors
- Ensure CORS is enabled on backend
