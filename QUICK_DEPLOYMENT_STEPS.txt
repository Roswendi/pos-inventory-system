METHOD 1: LOCAL NETWORK DEPLOYMENT (Easiest)
================================================================================

STEP 1: Prepare Server Computer
-------------------------------
1. Install Node.js (if not installed)
   Download from: https://nodejs.org
   Version: 14.x or higher

2. Copy pos-inventory-system folder to server computer

STEP 2: Install Dependencies
-----------------------------
Open Terminal/Command Prompt:

  cd pos-inventory-system/backend
  npm install

  cd ../frontend
  npm install

STEP 3: Build Frontend
-----------------------
  cd frontend
  npm run build

STEP 4: Update Backend to Serve Frontend
-----------------------------------------
Edit: backend/server.js

Add these lines BEFORE the routes section:

  const path = require('path');
  app.use(express.static(path.join(__dirname, '../frontend/build')));

Add this line AFTER all routes (before app.listen):

  app.get('*', (req, res) => {
    res.sendFile(path.join(__dirname, '../frontend/build/index.html'));
  });

STEP 5: Start Server
--------------------
  cd backend
  NODE_ENV=production node server.js

STEP 6: Find Server IP Address
------------------------------
Mac/Linux:
  ifconfig | grep "inet "

Windows:
  ipconfig

Look for: 192.168.x.x or 10.0.x.x

STEP 7: Access from Client Devices
-----------------------------------
On ANY device (computer, tablet, iPad):
1. Connect to same WiFi network
2. Open browser
3. Go to: http://[SERVER_IP]:5001
   Example: http://192.168.1.100:5001
   
   ‚ö†Ô∏è IMPORTANT: Use port 5001, NOT 3000!
   - Port 3000 is for development only
   - Port 5001 is for client deployment
   - Backend serves both API and frontend on port 5001

4. Login with:
   Username: admin
   Password: admin123

STEP 8: Install on Tablets/iPads (PWA)
---------------------------------------
iPad (Safari):
1. Open Safari
2. Go to: http://[SERVER_IP]:5001
3. Tap Share button (square with arrow)
4. Tap "Add to Home Screen"
5. Name it "TDO POS"
6. Tap "Add"

Android Tablet (Chrome):
1. Open Chrome
2. Go to: http://[SERVER_IP]:5001
3. Tap menu (3 dots)
4. Tap "Add to Home screen"
5. Confirm

Now it works like a native app!

================================================================================
METHOD 2: CLOUD DEPLOYMENT (For Multiple Locations)
================================================================================

BACKEND: Deploy to Railway
---------------------------
1. Sign up: https://railway.app
2. New Project ‚Üí Deploy from GitHub
3. Select your repository
4. Add environment variables:
   PORT=5001
   JWT_SECRET=your-secret-key
   NODE_ENV=production
5. Deploy
6. Copy your Railway URL (e.g., https://your-app.railway.app)

FRONTEND: Deploy to Netlify
----------------------------
1. Sign up: https://netlify.com
2. New site ‚Üí Deploy from GitHub
3. Build settings:
   Base directory: frontend
   Build command: npm install && npm run build
   Publish directory: frontend/build
4. Environment variables:
   REACT_APP_API_URL=https://your-app.railway.app/api
5. Deploy
6. Copy your Netlify URL (e.g., https://your-app.netlify.app)

CLIENT ACCESS:
--------------
- Any device with internet
- Go to: https://your-app.netlify.app
- Works on: Computers, iPads, Android tablets, phones

AUTOMATIC STARTUP (Optional)
================================================================================

Install PM2 for auto-start:
  npm install -g pm2
  cd backend
  pm2 start server.js --name "pos-backend"
  pm2 save
  pm2 startup

Server will start automatically on boot!

================================================================================
TROUBLESHOOTING
================================================================================

Can't access from client device?
- Check server is running
- Verify IP address
- Check firewall settings
- Ensure same WiFi network
- Try different browser

Slow performance?
- Check server RAM/CPU
- Clear browser cache
- Check network speed

Connection errors?
- Verify backend running
- Check port 5001 open
- Test: http://[SERVER_IP]:5001/api/health
