================================================================================
    PORT EXPLANATION: 3000 vs 5001
================================================================================

‚úÖ ANSWER: For Client Deployment, Use Port 5001 ONLY!
================================================================================

PORT 3000 - Development Only (NOT for Clients):
-------------------------------------------------
What it is:
  - React development server
  - Used ONLY during development/testing
  - Hot reload enabled
  - Runs separately from backend

When to use:
  - When developing/testing the app
  - When you want hot reload
  - NOT for client deployment

Command:
  cd frontend
  npm start
  # Runs on http://localhost:3000

Access:
  - Only from localhost (your computer)
  - Not easily accessible from other devices
  - Requires backend to run separately on port 5001

================================================================================

PORT 5001 - Production/Client Deployment (USE THIS):
-----------------------------------------------------
What it is:
  - Backend API server
  - In production mode, also serves the frontend
  - Single server for everything
  - Optimized for client use

When to use:
  - For client deployment ‚úÖ
  - For production use ‚úÖ
  - When clients need to access the app ‚úÖ

Command:
  cd backend
  NODE_ENV=production node server.js
  # Runs on http://localhost:5001

Access:
  - From any device on network: http://[SERVER_IP]:5001
  - Serves both API and frontend
  - Single URL for everything

================================================================================

HOW IT WORKS:
=============

DEVELOPMENT MODE:
-----------------
Frontend (Port 3000):
  - React development server
  - http://localhost:3000
  - Connects to backend API at port 5001

Backend (Port 5001):
  - API server only
  - http://localhost:5001/api
  - Provides data to frontend

Two separate servers running!

PRODUCTION MODE:
----------------
Backend (Port 5001):
  - Serves API: http://[IP]:5001/api
  - Serves Frontend: http://[IP]:5001
  - Everything on one port!

Single server, one port!

================================================================================

FOR CLIENT DEPLOYMENT:
======================

STEP 1: Build Frontend
-----------------------
  cd frontend
  npm run build

STEP 2: Start Production Server
-------------------------------
  cd backend
  NODE_ENV=production node server.js

STEP 3: Client Access
---------------------
Clients use: http://[SERVER_IP]:5001

NOT port 3000!

================================================================================

EXAMPLE:
========

Server IP: 192.168.1.100

Development (Your Testing):
  - Frontend: http://localhost:3000
  - Backend:  http://localhost:5001/api

Production (Client Use):
  - Everything: http://192.168.1.100:5001
  - API:        http://192.168.1.100:5001/api
  - Frontend:   http://192.168.1.100:5001

================================================================================

QUICK REFERENCE:
================

Development:
  - Frontend: npm start (port 3000)
  - Backend:  node server.js (port 5001)
  - Two servers

Production/Client:
  - Build: npm run build
  - Start: NODE_ENV=production node server.js (port 5001)
  - One server, one port

Clients access: http://[IP]:5001
