3. STEP 1: CREATE APP ICONS (5 minutes)
================================================================================

You need to create icon files for the app to work properly on mobile devices.

REQUIRED FILES:
---------------
Create these files in: frontend/public/
- logo192.png (192x192 pixels)
- logo512.png (512x512 pixels)
- favicon.ico (optional but recommended)

METHOD 1: ONLINE ICON GENERATOR (Easiest)
------------------------------------------
1. Go to: https://www.favicon-generator.org/
2. Upload your logo/image (square image works best)
3. Click "Generate Favicons"
4. Download the package
5. Extract and copy:
   - android-chrome-192x192.png â†’ rename to logo192.png
   - android-chrome-512x512.png â†’ rename to logo512.png
6. Place both files in: frontend/public/

METHOD 2: USING IMAGE EDITOR
------------------------------
1. Open your logo in Photoshop, GIMP, or Canva
2. Create a square canvas (512x512 pixels)
3. Center your logo
4. Export as PNG:
   - 192x192 pixels â†’ Save as logo192.png
   - 512x512 pixels â†’ Save as logo512.png
5. Place both files in: frontend/public/

METHOD 3: CREATE SIMPLE LOGO (If you don't have one)
-----------------------------------------------------
1. Go to https://www.canva.com (free)
2. Create a new design: 512x512 pixels
3. Add text "POS" or your business name
4. Choose colors (match your theme: #0066FF blue)
5. Export as PNG
6. Resize to both sizes:
   - 192x192 â†’ logo192.png
   - 512x512 â†’ logo512.png
7. Place both files in: frontend/public/

ICON DESIGN TIPS:
----------------
- Use square images (1:1 ratio)
- Simple designs work best at small sizes
- High contrast for visibility
- Match your brand colors
- Test at small sizes (192px) to ensure readability

VERIFY ICONS ARE CREATED:
-------------------------
Open terminal and run:
  cd frontend/public
  ls -la logo*.png

You should see both logo192.png and logo512.png

10. TROUBLESHOOTING
================================================================================

PROBLEM: Icons not showing
---------------------------
SOLUTION:
- Verify files exist: ls frontend/public/logo*.png
- Check file names are exactly: logo192.png and logo512.png
- Clear browser cache
- Rebuild: npm run build
- Redeploy frontend

PROBLEM: API not working / CORS errors
---------------------------------------
SOLUTION:
- Verify backend URL is correct in frontend/src/services/api.js
- Test backend directly in browser
- Check backend is running (visit backend URL)
- Verify CORS is enabled in backend/server.js
- Check browser console for specific error messages

PROBLEM: App won't install on mobile
-------------------------------------
SOLUTION:
- Must use HTTPS (not HTTP) - free hosting provides this
- Check manifest.json is accessible (visit: your-url/manifest.json)
- Check service worker is registered (browser console)
- Try different browser (Chrome for Android, Safari for iOS)
- Clear browser cache
- Check browser console for errors

PROBLEM: Build fails
---------------------
SOLUTION:
- Delete node_modules: rm -rf node_modules
- Delete package-lock.json: rm package-lock.json
- Reinstall: npm install
- Check Node.js version (need 14+): node --version
- Check for error messages in terminal
- Try: npm cache clean --force

PROBLEM: Backend deployment fails
----------------------------------
SOLUTION:
- Check Root Directory is set to "backend"
- Verify PORT environment variable is set
- Check deployment logs for errors
- Ensure package.json exists in backend folder
- Verify all dependencies are in package.json

PROBLEM: Frontend shows blank page
-----------------------------------
SOLUTION:
- Check browser console for errors
- Verify build completed successfully
- Check that index.html exists in build folder
- Verify API URL is correct
- Check network tab for failed requests

PROBLEM: Login doesn't work
---------------------------
SOLUTION:
- Verify backend is running
- Check API URL is correct
- Test backend login endpoint directly
- Check browser console for errors
- Verify default user exists (admin/admin123)

PROBLEM: Service Worker not registering
-----------------------------------------
SOLUTION:
- Verify sw.js exists in public folder
- Check sw.js is accessible (visit: your-url/sw.js)
- Must use HTTPS
- Check browser console for errors
- Clear browser cache
